Judging Criteria

How to Win! Or Not Win! You Decide.

Creativity

Innovation
  • The bot solves a real world problem
  • The bot solves a unique problem or is a novel approach to the problem
  • The bot is practical for the problem
Evocative
  • The bot evokes emotion (humor, excitement, relief, etc.)
  • Users desire to continue using the bot
  • Users are likely recommend the bot to a friend
Impact
  • Provides significant value to its users
  • The bot is a definitive solution to the problem it aims to solve

Usefulness

User Experience
  • Bot commands are intuitive and easy to use
  • Bot response time meets or exceeds user expectation
  • Internal and external presentation is consistent w/ bot's intended purpose
Documentation
  • Documentation is easy to follow and use
  • Technical documentation gives clear overview of how app is built and functions architecturally
  • User documentation gives clear instructions on intended use and purpose
Quality
  • The bot works as described in documentation
  • The bot functions from end-to-end w/o issues
  • (If Applicable) The bot follows third-party API guidelines

Requirements

You Should Probably Read This. Probably.

Documentation

All submissions will require technical and user documentation. Both can be included as part of a README document in your GitHub Repository.

Technical Documentation

Explain the design of your bot. Be sure to include details about system components, architecture, libraries, and external APIs.

User Documentation

Explain to an end-user how your bot works in a Discord channel. Provide examples and use cases!

GitHub Repository

Submissions require a public GitHub repository for source code review.

Remote Deployment

We want to test your bots. Deploying your bot on a remote server will make your bot available for us even when you're not there.

Official Rules

This challenge is open to current members of the Aggie Coding Club.

You need to prepare a document containing technical and user documentation when submitting your project. Technical documentation should include a high-level system overview that includes an architectural description of interactions between system components. User documentation should include information on how to use your bot in a discord channel, including any bot commands. Both sets of documentation may be included in a GitHub repository as a ReadMe.md file. You are free to include additional documentation, videos or images presenting your solution.

The submission and all its assets must be publicly available via a GitHub repository during the judging period. In order for your submission to be judged, your bot must be available to add to a Discord Server and functional during the judging period.

Please keep in mind that the submission for the Challenge must be a working project that acts as an automated Discord user (bot). Submissions must be original, the work of the participants, and must not infringe, misappropriate, or otherwise violate any IP (intellectual property) rights, proprietary rights, privacy rights, moral rights, or any other rights of any person or entity. Any case of plagiarism may result in disqualification of the participant from the Challenge. Submissions must be in English or translated accordingly.

Submissions must be newly created products that were worked on during the Fall 2020 semester of Texas A&M University.
